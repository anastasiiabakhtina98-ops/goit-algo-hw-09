## Порівняння ефективності алгоритмів видачі решти

### Оцінка складності

- **Жадібний алгоритм `find_coins_greedy`**  
  Для фіксованого набору монет алгоритм один раз проходить по масиву номіналів, на кожному кроці беручи максимально можливу кількість монет поточного номіналу. Часова складність становить \(O(n)\), де \(n\) — кількість різних номіналів монет, а для фіксованої грошової системи це фактично константа. Просторова складність дорівнює \(O(1)\) додатково до структури результату.

- **Алгоритм динамічного програмування `find_min_coins`**  
  Будується масив `dp` довжиною `amount + 1`, де для кожної суми від 0 до `amount` обчислюється мінімальна кількість монет, а також масив `last_coin` для відновлення рішення. Для кожної монети здійснюється прохід по всіх можливих проміжних сумах, тому часова складність дорівнює \(O(n * S)\), де \(n\) — кількість номіналів, а \(S\) — цільова сума. Просторова складність становить \(O(S)\) за рахунок масивів `dp` та `last_coin`.

### Поведінка на великих сумах

- Для великих сум жадібний алгоритм є дуже масштабованим: його час виконання практично не залежить від величини суми, а залежить лише від кількості номіналів монет. Це робить його особливо привабливим для касових систем, де операції мають виконуватися миттєво.
- Алгоритм динамічного програмування лінійно (або майже лінійно) зростає за часом і пам’яттю із збільшенням суми, оскільки необхідно обробити всі проміжні значення від 0 до `amount`. Для дуже великих сум це може призвести до значних витрат пам’яті та відчутного часу обчислення.

### Коли який алгоритм ефективніший

- Для «класичних» грошових систем (наприклад, номінали 1, 2, 5, 10, 25, 50) жадібний алгоритм завжди знаходить оптимальне за кількістю монет рішення, тому він є і швидшим, і простішим у реалізації, і достатньо точним для практичного використання в касових апаратах.
- Для довільних або «патологічних» наборів номіналів жадібний алгоритм може давати не мінімальну кількість монет. У таких випадках алгоритм динамічного програмування є більш надійним, оскільки гарантовано знаходить оптимальне рішення, хоч і ціною більшої часової та просторової складності.

У підсумку, для даної задачі з фіксованими номіналами монет касовій системі доцільно використовувати жадібний алгоритм як основний, а підхід на основі динамічного програмування — як універсальний резервний інструмент для нетипових наборів номіналів або аналізу найоптимальніших рішень.
